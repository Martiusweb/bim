Install Party GPL

Avant l'installation

Lancer la commande suivant : 

cd && wget http://192.168.2.1/pre_install.sh && chmod +x pre_install.sh && sudo ./pre_install.sh
Il est normal que parfois ça demande une confirmation (taper "Yes" par exemple), mais globalement aucune erreur ne doit s'afficher. Si c'est le cas, référez vous au paragraphe qui va bien.

Rebooter la machine sur l'OS installé

Après l'installation

Un autre script est disponible pour mettre à jour la distribution et installer les paquets essentiels au système. Il peut être interressant de montrer l'installation d'un paquet si l'utilisateur est néophyte, en utilisant la Logithèque Ubuntu. 

cd && wget http://192.168.2.1/post_install.sh && chmod +x post_install.sh && sudo ./post_install.sh
Il est normal que parfois ça demande une confirmation (taper "Yes" par exemple), mais globalement aucune erreur ne doit s'afficher. Si c'est le cas, référez vous au paragraphe qui va bien.

Si ça ne fonctionne pas

Vous ne vous énervez pas, vous faites à Paul un rapport de bug (quelle étape a planté, message d'erreur, config particulière de la machine, etc.). Vous lisez les scripts, et vous faites les étapes à la main, en adaptant. Il sont commentés comme il faut normalement.

Après tout ça

Verifier que le proxy APT n'est plus là

Le fichier /etc/apt/sources.list dois PAS contenir de lignes comportant l'IP du cache apt (192.18.2.1 pour rappel)..

Tour d'horizon du système

C'est le moment de montrer de quoi Linux est capable. Il faut demander aux utilisateurs s'ils ont des besoins spécifiques et tenter d'y répondre. Mettre l'accent sur la logithèque Ubuntu (le logiciel et le concept) et le site Ubuntu-fr et sur le site du zéro pour les développeurs et le tutoriel GNU/Linux.
